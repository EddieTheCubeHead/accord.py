Embed verification

utils.embed_helpers.EmbedField = tuple[str | None, str | None, bool] | tuple[str | None, str | None]

A type representing a tuple containing embed field data.

The first two fields represent the name and value of the field and are mandatory. Last field is optional and represents the inline-status of the tuple. First two fields are optional strings, the last field should be bool if present.

class utils.embed_helpers.EmbedVerifier(*, title: str | None = <utils.embed_helpers._NoneSentinel object>, description: str | None = <utils.embed_helpers._NoneSentinel object>, author_name: str | None = <utils.embed_helpers._NoneSentinel object>, author_icon_url: str | None = <utils.embed_helpers._NoneSentinel object>, author_url: str | None = <utils.embed_helpers._NoneSentinel object>, colour: int | None = <utils.embed_helpers._NoneSentinel object>, color: int | None = <utils.embed_helpers._NoneSentinel object>, footer_text: str | None = <utils.embed_helpers._NoneSentinel object>, footer_icon_url: str | None = <utils.embed_helpers._NoneSentinel object>, image_url: str | None = <utils.embed_helpers._NoneSentinel object>, thumbnail_url: str | None = <utils.embed_helpers._NoneSentinel object>, fields: ~typing.Iterable[tuple[str | None, str | None, bool] | tuple[str | None, str | None]] | None = <utils.embed_helpers._NoneSentinel object>)

A class for verifying embeds created with discord.py

You can verify that the embed is exactly identical to the constructed verifier object with matches_fully() or that all configured fields are identical with matches_configured().

Attention

All text-based fields are regex based.

If you need regex characters, please escape them. Disabling regex based validation should be possible at some point in the future.

Keyword Arguments:

  • title (str | None) – The regex validation match for the title of the embed.

  • description (str | None) – The regex validation match for the description of the embed.

  • author_name (str | None) – The regex validation match for the name of the author of the embed.

  • author_icon_url (str | None) – The regex validation match for the icon url of the author of the embed.

  • author_url (str | None) – The regex validation match for the url of the author of the embed.

  • colour (int | None) – Numeric validation match for the hex-based colour of the embed.

  • color (str | None) – US-english based alias for colour

  • footer_text (str | None) – The regex validation match for the footer text of the embed.

  • footer_icon_url (str | None) – The regex validation match for the icon url of the footer of the embed.

  • image_url (str | None) – The regex validation match for the image url of the embed.

  • thumbnail_url (str | None) – The regex validation match for the thumbnail url of the embed.

  • fields (Iterable[EmbedField]) – An iterable containing EmbedField tuples representing the fields of the embed.

matches_configured(embed: Embed, *, allow_extra_fields: bool = False, allow_any_field_order: bool = False)

A method to verify given embed matches the configured fields of the EmbedVerifier.

This means all fields not explicitly set are ignored for verification purposes.

Parameters:

embed – The embed to verify.

Keyword Arguments:

  • allow_extra_fields – Whether more fields are allowed to be present than originally configured. If fields are not configured, any number of fields are accepted. Defaults to False

  • allow_any_field_order – Whether the configured fields should be allowed to exist in any order. Defaults to False

matches_fully(embed: Embed)

A method to verify given embed fully matches the configuration of the EmbedVerifier.

This means all fields not explicitly set are expected to be None for verification purposes.

Parameters:

embed – The embed to verify.
