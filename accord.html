exception accord.AccordException

Exception thrown by accord.py, subclasses Exception with no interface changes

class accord.Engine(client: Client, command_tree: CommandTree)

The engine that is responsible for running commands and events against the client under test.

Caution

You should not instantiate accord.Engine manually, instead the accord.create_engine() method should be used.

client

The client under test

command_tree

The command tree of the client. Can be None if you are not testing application commands.

async app_command(command_name: str, *args, command_guild: int | Guild = None, issuer: int | User = None, channel: int | TextChannel = None, **kwargs)

A coroutine to send an application command to the client.

Attention

Accepts *args and **kwargs to be sent to the command.

Raises:

accord.AccordException – on guild/channel mismatch or if channel is not given and guild does not have a default channel

Parameters:

command_name – The name of the command to be run

Keyword Arguments:

  • command_guild – The guild or the id of the guild the command should be run on. None uses the default guild (see guild). Defaults to None

  • issuer – The user or the id of the user issuing the command. None uses the default member (see member). Defaults to None

  • channel – The channel of the id of the channel the command is issued on. None uses the default text channel (see text_channel). Defaults to None

property button: Button | None

A property returning the first button of the view associated with the response, if available

Attention

async accord.create_engine(client: Client, command_tree: CommandTree) → Engine

A method to create an accord.Engine instance to be used with testing.

Caution

You should not instantiate accord.Engine manually, instead this method should be used.

Parameters:

  • client – The client that the engine should run against

  • command_tree – The command tree of the client

Returns:

An engine instance that can be used to run commands or events on the client

accord.create_guild(name: str = None, create_default_channel: bool = True) → Guild

A method for creating a new mock guild.

Caution

You should not instantiate discord_objects.Guild manually, instead this method should be used.

Attention

Guild id is generated automatically

Parameters:

  • name – The name of the guild to be created. None uses Guild {id}. Defaults to None

  • create_default_channel – Whether to create a default text channel for the guild. Defaults to True

Returns:

The created discord_objects.Guild object

accord.create_text_channel(channel_guild: int | Guild = None, name: str = None) → TextChannel

A method for creating a new mock channel

Caution

You should not instantiate discord_objects.TextChannel manually, instead this method should be used.

Attention

TextChannel id is generated automatically

Parameters:

  • channel_guild – The guild object or the id of the guild the channel should belong to. None uses the default guild (see: guild). Defaults to None

  • name – The name of the text channel to be created. None uses Text channel {id}. Defaults to None

Returns:

The created discord_objects.TextChannel object

accord.create_user(name: str = None, avatar: str = None, discriminator: str = None)

A method for creating a new mock user

Caution

You should not instantiate discord_objects.User manually, instead this method should be used.

Attention

User id is generated automatically

Parameters:

  • name – The name of the user to be created. None uses User {id}. Defaults to None.

  • avatar – The avatar url of the user to be created. None uses User {id} avatar. Defaults to None.

  • discriminator – The discriminator of the user to be created. None uses {id}. Defaults to None.

Returns:

The created discord_objects.User object
